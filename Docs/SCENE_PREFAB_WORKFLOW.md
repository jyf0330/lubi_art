# 根目录地点场景与独立预制体工作流

## 结构

根目录 `Scenes/` 是美术打开项目后直接进入的地点场景目录。一个地点场景由多个独立预制体拼装，而不是一个预制体对应一个场景文件夹。

```text
Scenes/
├── MainMenu/MainMenuScene.tscn
├── ArtistFlow/ArtistFlowScene.tscn
└── Battle/BattleScene.tscn

每个地点场景
└── 实例化多个 Features/*/Prefabs 或 Shared/Prefabs
    ├── 场景保存各预制体在该地点的整体位置
    └── Prefab 保存自己的内部节点、尺寸、资源、动画和公开脚本
```

`Features/*/Views/*.tscn` 只保留稳定运行时入口，实例化对应的根目录地点场景，避免程序、测试和迁移路径因美术目录调整而变化。

## 美术怎么改

### 调整某个预制体在地点中的位置

直接打开 `Scenes/<Location>/<Location>Scene.tscn`，移动预制体实例的根节点。这个坐标只属于当前地点，不同步到通用预制体。

### 调整预制体内部美术

1. 在地点场景中找到该预制体实例。
2. 打开它对应的 `Prefabs/*.tscn` 源文件。
3. 在独立预制体中修改内部图片、Label、按钮、边框、尺寸、间距或动画。
4. 保存独立预制体后，所有地点场景中的实例自动同步。
5. 地点场景不得长期保存 `Editable Children` 产生的内部覆盖。

如果美术先在地点场景里试出了内部调整结果，交付前必须把同样的内部修改应用到独立预制体，并清除地点实例上的内部覆盖。AI 修改场景内预制体时也必须同时修改对应独立预制体，不能只改场景副本。

## 数据归属

```text
地点场景保存
├── 预制体实例的整体 position / anchors / offsets
├── 只在该地点成立的整体 scale、z_index 和 visible
└── 多个预制体之间的空间关系

独立预制体保存
├── 内部节点相对位置和尺寸
├── 内部图片、文字、边框与状态
├── 动画轨道和内部层级
├── 公开表现脚本
└── 组件自身最小尺寸
```

## 验收

运行 `Tests/Prefabs/location_scene_prefab_sync_smoke.gd`。测试会确认：

- 三个根目录地点场景都存在；
- ArtistFlow 和 Battle 确实由多份独立预制体组成；
- 稳定 View 没有复制地点场景节点；
- 地点场景没有保存预制体内部子节点覆盖；
- 实例化后的稳定 View 与根目录地点场景结构一致。
